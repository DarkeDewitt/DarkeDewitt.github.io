          # 前言

          这是一个使用 Vuepress 创建的博客，而这片文章将是一切的开始。

          经过了两个多月的春招，发现了自己很多的不足，所以想要创建这个博客，记录下自己每天的成长，从各处学到的新知识，读书心得，面试遇到的问题之类的，都会放在这上面。

          之前也有用 hexo 在 Git pages 上创建了一个博客，不过那个博客我好像只写了一篇文章记录我在看红宝书的心得（汗Σ( ° △ °|||)︴），一切都要归结于我太懒了吧，没有这种记录的习惯。希望能在新的博客上坚持记录和分享。

          # 关于 Vuepress

          首先 Vuepress 是由尤雨溪大大在一个月前开发并维护的开源项目，特点是可以用.md来写文章，也可以用 vue 来创建博客页面。最近用 vue 用的比较多，所以看到这个项目就很亲切，感觉能有比较好的体验，而且这也能算一个练手项目吧（真的吗我相当怀疑）哈哈哈。

          # 创建过程

          1. 首先，当然是根据官网的文档一步步来啦。由于是尤雨溪大大的项目，所以详细的中文文档也是标配（这样好.jpg）。

          2. 在本地创建好一个文件夹，用来装我们这个博客项目（这里你可以选择先从github创建好再clone下来），记得先npm init一下，然后安装好vuepress。

          WARNING

          官网的例子中有这么一个语句：

          echo "# Hello VuePress!" > docs/README.md

          我在实际创建的过程中呢，系统是win10，使用的是powershell，这时候产生的.md文件呢，编码可能不是utf-8（我自己创建出来的就是utf-16），这样在预览的时候你就会发现，怎么首页变成乱码了呀，这时候使用编辑器查看一下你的.md文件的编码，修改一下就好。（当然最直接的办法就是不要用这个语句来生成.md文件)

          1. 然后就是部署到 github 上面啦。创建一个 自己 id.github.io 的仓库，然后查看文档，copy里面的代码，到项目中新建一个 deploy.sh 文件，按照文档操作，然后用gitbash等工具运行这个.sh文件就成功部署啦！

          TIP

          可能有的同学在本机上第一次连 github，没有生成自己的 ssh key 绑定到 github 上，运行就报了错。这时候在 github 的 setting 里面找到 ssh and GPG keys 这一项，在ssh keys下参照文档将自己的 key 添加到 github 账户上就ok啦（没有阅读文档能力的同学，需要好好学习了）。
